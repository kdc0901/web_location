## 3. `git reset` — 장단점

### 하는 일

지정한 커밋 시점으로 **브랜치 끝(HEAD)** 을 옮기고, 파일 내용도 그때와 같게 맞춥니다.

### 미니맵 직전으로 (로컬만, 확실히 되돌리기)

```bash
cd "/Volumes/Exe, Todo(md,book,app)_2T mac/_location"
git reset --hard 4aee493
```

| 장점 | 단점 |
|------|------|
| 한 번에 모든 파일이 **그 커밋과 동일** | `--hard`는 **그 이후 커밋·미커밋 변경 삭제** |
| 빠름, 실수(파일 빠뜨림) 적음 | 이미 `git push` 했으면 원격과 **어긋남** |

### 이미 GitHub에 push 했을 때

로컬만 `reset --hard` 하면 원격에는 여전히 `ef7c7fa`가 있습니다.  
**혼자 쓰는 저장소가 아니면** `git push --force`는 피하고, 아래 **revert** 를 권장합니다.

## 4. `git revert` — push 후에 더 안전

**「미니맵 추가」 커밋만 취소**하는 **새 커밋**을 만듭니다. 히스토리는 지우지 않습니다.

```bash
cd "/Volumes/Exe, Todo(md,book,app)_2T mac/_location"
git revert ef7c7fa --no-edit
git push origin main
```

| 장점 | 단점 |
|------|------|
| GitHub Pages·협업에 **안전** | 커밋이 하나 더 쌓임 |
| “무엇을 되돌렸는지” 기록이 남음 | 충돌 나면 해결 필요할 수 있음 |

**이미 `ef7c7fa`를 push 했다면 → `revert`가 일반적으로 가장 무난합니다.**

## 5. Cursor에서 「이전 기능으로 돌려줘」 — 장단점

### 잘 맞는 경우

- **미니맵만** 빼고, **찾기 → 전체배치도** 레이아웃은 **유지**
- **어느 시점**으로 갈지 아직 정하지 못함 → 대화로 단계 조절
- **문서만** 수정·정리
- Git을 쓰지 않거나 커밋이 없음

### 주의할 점

- AI가 파일을 **다시 작성**하므로, `4aee493`과 **바이트 단위로 100% 같다고 보장하기 어렵음**
- `style.css?v=6`, `sw.js` 캐시 이름 등 **일부만** 남을 수 있음
- 여러 파일(`index.html`, `app.js`, `css`, `sw.js`) 중 **하나 빠뜨리면** 화면이 깨질 수 있음

### Cursor에 요청할 때 (효율적인 예시)

모호한 요청보다 **구체적**일수록 좋습니다.

- ❌ 「예전으로 돌려줘」
- ✅ 「`ef7c7fa` 미니맵 추가 이전처럼, 검색 결과는 이름+위치 텍스트만, `4aee493` 상태에 맞게 `index.html` `app.js` `style.css` `sw.js` 수정」

## 6. 방법별 비교표

| 기준 | `git reset --hard` | `git revert` | Cursor 되돌리기 |
|------|-------------------|--------------|-------------------|
| 속도 | 매우 빠름 | 빠름 | 보통~느림 |
| 이전 커밋과 동일 | ◎ | ◎ (revert 결과) | △ |
| push 후 안전 | △ (force 필요) | ◎ | 해당 없음 |
| 일부만 변경 | ✕ | △ | ◎ |
| Git 기록 | 이후 커밋 사라짐(로컬) | 유지 | 파일만 변경 |

## 9. 자주 묻는 질문

### Q. `git reset` 하면 파일이 영구 삭제되나?

`--hard`는 **그 커밋 이후 변경을 working tree에서 지웁니다.**  
이미 push된 커밋은 원격에 남아 있고, `reflog`로 복구할 여지도 있습니다.  
**확실하지 않으면** `git revert` 또는 reset 전에 `git branch backup-before-reset` 으로 브랜치 하나 만드세요.

### Q. Cursor만 쓰면 Git 안 배워도 되나?

당장 되돌리기는 가능하지만, **언제가 어떤 코드였는지** 추적은 Git이 훨씬 낫습니다.  
이 프로젝트는 이미 커밋이 쌓여 있으므로 **Git 사용을 권장**합니다.

### Q. 문서(`찾은 후 전체배치미니맵추가.md`)는?

코드 되돌려도 **문서 파일은 그대로** 남을 수 있습니다.  
미니맵을 쓰지 않을 거면 문서는 삭제하거나 「사용 안 함」 표시해 두면 됩니다.
