# 반응형 아직, 재요청

PC에서는 보였지만 **휴대폰·카카오톡 링크**에서는 레이아웃이 맞지 않아, 반응형을 **다시 요청**했을 때의 요구 사항과 처리 내용을 정리한 문서입니다.

관련 문서:
- `반응형으로 개선.md` — 실제 적용된 개선 내용
- `6_2) (자리배치 파일)을 GitHub에 배포하는 절차.md` — GitHub Pages 배포

---

## 1. 왜 재요청했는가 (아직 부족했던 점)

| 구분 | 이전 상태 | 문제 |
|------|-----------|------|
| 화면 순서 | 제목 → 검색 → 배치도 (결과가 검색창 안에만 표시) | **이름·위치**가 눈에 잘 안 띔 |
| 모바일 두레 | 2열 또는 좁은 3열 | 폰에서 칸·글자가 작고 읽기 어려움 |
| 배치도 위치 | 검색과 배치도 사이 구분 약함 | 첨부 이미지와 다른 흐름 |
| 이름 칸 | `(중등) 이현주` 등 **...** 으로 잘림 | 긴 이름 확인 불가 |
| 하단 1명 칸 | 김백수·김지은 칸만 가로로 길게 늘어남 | 다른 칸과 크기 불일치 |

→ **「모바일에서도 잘 보이게 + 첨부 이미지 레이아웃」** 으로 **재요청**.

---

## 2. 재요청 원문 (요구 사항)

> 모바일(휴대폰)에서도 잘 보이려면 반응형으로도 구현해주고,  
> 레이아웃은 첨부이미지대로  
> - 이름 찾기 : 상단에 두고  
> - 전체 배치도를 이름찾기 바로 아래에  
> - 해당이름과 위치 보여주기  

### 첨부 이미지 기준 레이아웃

| 영역 | 내용 |
|------|------|
| 상단 | 제목, 부제(8두레 · 38명) |
| 검색 | 「이름으로 자리 찾기」 입력 + 찾기 |
| **결과** | 검색한 사람 이름 + 위치 (또는 배치도 **노란 강조**) |
| 배치도 | 승현반 칠판 → 8두레 그리드 |
| PC | 두레 **한 줄에 3개** |
| 모바일 | 두레 **세로 1열**, 칸·이름 크게 |

## 3. 재요청 대응 체크리스트

| # | 요구 | 처리 | 파일 |
|---|------|------|------|
| 1 | 이름 찾기 상단 | `top-section` + 모바일 sticky | `index.html`, `style.css` |
| 2 | 배치도를 검색 바로 아래 | `classroom`을 `top-section` 다음에 배치 | `index.html` |
| 3 | 이름·위치 표시 | `result-card` (찾은 사람 / 위치) | `index.html`, `app.js` |
| 4 | 검색 시 배치도 강조 | 책상·두레 **노란색** highlight | `app.js`, `style.css` |
| 5 | 모바일 1열 | `@media (max-width: 640px)` 1열 | `style.css` |
| 6 | 태블릿 2열 | `@media (max-width: 900px)` 2열 | `style.css` |
| 7 | PC 3열 | 기본 `grid-template-columns: repeat(3, 1fr)` | `style.css` |
| 8 | 이름 잘림 제거 | `ellipsis` 제거, 줄바꿈 허용 | `style.css` |
| 9 | 하단 1명 칸 너비 통일 | `desk-row--single` 분리 | `app.js`, `style.css` |
| 10 | 휴대폰 터치 UI | 버튼 전체 너비, input 16px | `style.css` |

## 4. 재요청 후 화면 순서 (최종)

```
1. 교실 자리배치 (제목)
2. 8두레 · 총 38명
3. ┌ 이름으로 자리 찾기 ─┐
   │ [입력] [찾기]      │
4. ┌ 검색 결과 카드 ────┐  ← 이름 + 위치
5. ┌ 전체 배치도 ───────┐
   │ 칠판               │
   │ (가)(나)(다) ...    │
6. ▶ 그룹 위치 안내 (접기)
```

## 5. 아직 확인이 필요한 것 (배포·실기기)

코드는 반영되었어도, **아래를 하지 않으면 폰에서 예전 화면**이 보일 수 있습니다.

| 확인 항목 | 방법 |
|-----------|------|
| GitHub push 여부 | `git status` → 변경 있으면 commit & push |
| Pages 반영 대기 | push 후 1~3분 |
| 브라우저 캐시 | 강력 새로고침 `Cmd + Shift + R` |
| 올바른 URL | https://kdc0901.github.io/web_location/ |
| 실제 휴대폰 | 카톡 링크로 열어서 1열·결과 카드 확인 |

### GitHub 반영 명령어

```bash
cd "/Volumes/Exe, Todo(md,book,app)_2T mac/_location"
git add index.html css/ js/
git commit -m "반응형 재요청 반영: 검색결과 카드·모바일 1열"
git push origin main
```

## 6. 재요청 후에도 안 맞을 때

| 증상 | 점검 |
