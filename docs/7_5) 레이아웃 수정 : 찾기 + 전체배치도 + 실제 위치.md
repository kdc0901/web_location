# 7_5) 레이아웃 수정 : 찾기 + 전체배치도 + 실제 위치

교실 자리배치 웹앱에서 **「찾기」 버튼 바로 아래**에 **「전체배치도」** 제목을 두고, 그 아래에 **칠판 + 8개 두레 실제 배치**가 이어지도록 화면 순서를 바꾼 내용을 정리한 문서입니다.

관련 문서:
- `7_1) 반응형으로 개선.md` — 1차 반응형
- `7_2) 반응형 아직, 재요청.md` — 모바일 재요청 배경
- `7_3) 2차요청후 제대로 반영.md` — 모바일 우선(v3)
- `7_4) 제대로 반영후 홈피 배포때 url 주소 끝에 v3를 붙이는 이유(휴대폰 캐시 삭제).md` — 캐시 우회
- `6_2) (자리배치 파일)을 GitHub에 배포하는 절차.md` — 배포

프로젝트 폴더: `/Volumes/Exe, Todo(md,book,app)_2T mac/_location`

---

## 1. 요청 내용 (원문 요약)

| 순서 | 화면 요소 |
|------|-----------|
| 1 | 이름 입력 + **「찾기」** 버튼 |
| 2 | **「전체배치도」** 제목 |
| 3 | 칠판(12시 방향) + **8개 두레** 그리드 (실제 자리 배치) |

검색 전에는 **2번과 3번이 바로 이어져** 보이도록 하고, 검색 결과 카드는 **전체배치도와 배치도 사이에 끼지 않게** 처리했습니다.

---

## 2. 변경 전 vs 변경 후

### 변경 전 (문제)

- 전체 배치 영역이 `.classroom` 안에 있었고, 제목은 CSS `::before`로 **「전체 배치도」**만 표시됨
- 검색 결과 카드가 배치도 **위·아래**에 섞여 있어, 첨부 이미지와 같은 **「찾기 → 전체배치도 → 배치도」** 흐름이 약함

### 변경 후 (v4 레이아웃)

| 구분 | 내용 |
|------|------|
| 검색 | `top-section` — 이름 찾기 + 찾기 버튼 |
| 검색 결과 | 같은 `top-section` 안, **검색할 때만** 표시 (`hidden`) |
| 전체 배치 | `layout-section` — **「전체배치도」** `h2` + 칠판 + `#groups-grid` |

## 4. HTML 구조 (`index.html`)

핵심만 요약합니다.

```html
<div class="top-section">
  <section class="search-panel">… 입력 + 찾기 …</section>
  <div id="result-card" hidden>… 검색 결과 …</div>
</div>

<section class="layout-section">
  <h2 class="layout-title">전체배치도</h2>
  <div class="layout-map">
    <div class="blackboard">승현반 칠판 / 12시 방향</div>
    <div class="groups-grid" id="groups-grid"></div>
  </div>
</section>
```

| 클래스 / id | 역할 |
|-------------|------|
| `.top-section` | 검색 + (필요 시) 결과 카드 |
| `#result-card` | 찾기 후 이름·위치 표시, **평소 `hidden`** |
| `.layout-section` | 전체배치도 블록 전체 (흰 카드 배경) |
| `.layout-title` | **「전체배치도」** 글자 제목 |
| `.layout-map` | 칠판 + 두레 그리드 묶음 |
| `#groups-grid` | JavaScript가 8두레를 그리는 곳 |

## 5. CSS 변경 (`css/style.css`)

| 이전 | 이후 |
|------|------|
| `.classroom` + `::before { content: "전체 배치도" }` | `.layout-section` + `.layout-title` (HTML 제목) |
| (없음) | `.layout-map` — 배치도 영역 너비 100% |

스타일시트 버전: `css/style.css?v=4` (`index.html` 링크)

반응형(두레 1열·2열·3열) 규칙은 **7_3**과 동일하게 유지됩니다.

| 화면 너비 | 두레 열 수 |
|-----------|------------|
| 기본(모바일) | 1열 |
| 768px 이상 | 2열 |
| 1024px 이상 | 3열 |

## 6. JavaScript (`js/app.js`)

`setResultCard()` 동작:

| 상태 | `result-card` |
|------|----------------|
| `idle` (검색 전) | `hidden = true` → **전체배치도와 붙어 보임** |
| `success` | 카드 표시 + 배치도에서 해당 자리 **노란색** 강조 |
| `error` | 카드에 오류 메시지 |

검색 성공 시 `scrollIntoView`로 해당 **책상(desk)** 으로 스크롤합니다.
