# 반응형으로 개선

교실 자리배치 웹앱을 **PC·태블릿·휴대폰**에서 모두 보기 쉽게 반응형으로 개선한 내용을 정리한 문서입니다.

관련 문서: `6_2) (자리배치 파일)을 GitHub에 배포하는 절차.md`

프로젝트 폴더: `/Volumes/Exe, Todo(md,book,app)_2T mac/_location`

---

## 1. 개선 목표

| 목표 | 설명 |
|------|------|
| 레이아웃 순서 | 이름 찾기 → 검색 결과 → 전체 배치도 |
| 모바일 가독성 | 한 줄에 두레 1개, 터치하기 쉬운 버튼 |
| 검색 결과 강조 | 이름·위치를 카드로 명확히 표시 |
| 배치도 연동 | 검색 시 해당 자리 **노란색** 강조 |

## 3. 반응형 두레 배치

| 화면 너비 | 두레 그리드 | 예시 기기 |
|-----------|-------------|-----------|
| 901px 이상 | **3열** | PC, 넓은 태블릿 |
| 641px ~ 900px | **2열** | 태블릿 세로 |
| 640px 이하 | **1열** | 휴대폰 |

CSS 미디어 쿼리 위치: `css/style.css` 하단

```css
@media (max-width: 900px) { /* 2열 */ }
@media (max-width: 640px) { /* 1열 + 모바일 전용 */ }
```

## 4. 모바일 전용 기능

### 4-1. 상단 영역 고정 (sticky)

- `이름 찾기` + `검색 결과`가 스크롤 시 **상단에 붙음**
- 배치도만 아래로 스크롤

### 4-2. 입력·버튼

- 입력창 `font-size: 16px` → iOS에서 자동 확대 방지
- `type="search"`, `enterkeyhint="search"` → 휴대폰 키보드 최적화
- **찾기** 버튼 전체 너비 (세로 배치)

### 4-3. 두레·이름 칸

- 두레당 **3-2 배치** 유지 (위 3명, 아래 2명)
- 이름 전체 표시 (`...` 잘림 제거)
- 4명 두레 하단 1명 칸 = 위 칸과 **동일 너비**

### 4-4. 안전 영역

- `viewport-fit=cover`, `safe-area-inset` → 노치·홈 바 대응

## 5. 검색 결과 카드

| 상태 | 표시 |
|------|------|
| 대기 (`idle`) | "이름을 입력하고 「찾기」를 누르면…" |
| 성공 (`success`) | **찾은 사람** + **위치** (초록 배경) |
| 실패 (`error`) | 오류 메시지 (빨간 배경) |

위치 예시:

`(다) 두레 (중상 좌측) · 3번 자리 (1행 오른쪽)`

배치도에서 해당 책상은 **노란색**으로 강조됩니다.
