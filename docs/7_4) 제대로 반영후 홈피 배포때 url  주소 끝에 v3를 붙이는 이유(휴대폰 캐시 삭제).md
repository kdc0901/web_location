# 제대로 반영 후 홈피 배포 때 URL 주소 끝에 v3를 붙이는 이유

교실 자리배치를 **모바일 반응형으로 제대로 수정**한 뒤 GitHub Pages에 올렸는데도, 폰에서는 **예전 화면**이 보일 때가 있습니다.  
그때 URL 끝에 **`?v=3`** 을 붙여 접속하는 이유를 정리한 문서입니다.

관련 문서: `제대로 반응.md`

---

## 1. 기본 주소 vs v3 붙인 주소

| 구분 | URL |
|------|-----|
| 기본 | `https://kdc0901.github.io/web_location/` |
| v3 붙임 | `https://kdc0901.github.io/web_location/?v=3` |
| index 명시 | `https://kdc0901.github.io/web_location/index.html?v=3` |

**페이지 내용은 같습니다.**  
`?v=3`는 “이 주소는 **새로 받아라**”라고 브라우저에게 알리는 **표시**입니다.

---

## 2. 왜 v3를 붙이나? (한 줄)

> **서버는 이미 새 파일인데, 내 폰·브라우저만 예전 CSS/JS를 저장해 둔 채 쓰기 때문**

이걸 **캐시(Cache)** 문제라고 부릅니다.

---

## 3. 쉬운 비유

| 비유 | 설명 |
|------|------|
| 책 개정 | 출판사(GitHub)는 **새판**을 냈는데 |
| 내 책장 | 내 폰 브라우저는 **옛날 책**을 그대로 읽는 것 |
| `?v=3` | “**개정 3판** 주세요”라고 다시 주문하는 것 |

같은 홈페이지 주소라도, 끝에 `?v=3`을 붙이면 브라우저는 **다른 주소**로 인식해서 **서버에서 다시 받아옵니다.**

---

## 4. 코드 안에서도 v3를 쓴 곳

배포할 때 파일 안에도 **버전 3**을 넣어 두었습니다.

### 4-1. CSS 파일 주소 (`index.html`)

```html
<link rel="stylesheet" href="css/style.css?v=3" />
```

- `style.css`만 바꿔도 브라우저는 “예전에 받은 CSS”를 쓸 수 있음  
- `?v=3`을 붙이면 **새 CSS 파일**로 다시 요청

### 4-2. Service Worker 캐시 이름 (`sw.js`)

```javascript
const CACHE = "classroom-seating-v3";
```

- PWA·오프라인용으로 예전 HTML/CSS를 저장해 둠  
- 이름을 `v1` → `v3`로 바꾸면 **옛 캐시는 버리고** 새 파일을 저장

---

## 5. URL 끝 `?v=3` vs 파일 안 `?v=3`

| 위치 | 누가 쓰나 | 목적 |
|------|-----------|------|
| `css/style.css?v=3` (코드) | 개발자가 HTML에 작성 | CSS **항상** 새로 받기 |
| `...web_location/?v=3` (접속 URL) | **사용자·카톡 공유** | HTML·전체 페이지 **한 번** 새로 받기 |

**둘 다 같은 목적(캐시 끊기)** 이지만:

- 코드의 `?v=3` → push만 하면 대부분 자동 반영  
- URL의 `?v=3` → **직접 접속·공유할 때** 확실히 확인할 때 사용

---

## 6. 숫자 3의 의미

- **프로그램 3.0** 같은 공식 버전이 **아님**
- “반응형 **제대로** 고친 뒤, **세 번째**로 캐시를 갈아엎자”는 **구분용 번호**
- 다음에 또 크게 바꾸면 `?v=4`, `v4`로 올리면 됨

| 대략적 순서 | 내용 | 캐시 |
|-------------|------|------|
| v1 | 처음 GitHub Pages 배포 | `classroom-seating-v1` |
| (v2 미사용) | 1차 반응형 시도 | — |
| **v3** | 모바일 우선 + Grid + 겹침 수정 | `classroom-seating-v3` |

---

## 7. 언제 URL에 `?v=3`을 붙이나?

| 상황 | 권장 |
|------|------|
| push 직후 본인 폰에서 확인 | `?v=3` 붙여서 접속 |
| 카톡으로 “새로 고쳤어요” 공유 | `https://kdc0901.github.io/web_location/?v=3` |
| 이미 잘 보이는 사람 | 기본 URL만 써도 됨 |
| 여전히 옛 화면(겹침·잘림) | `?v=3` + 시크릿 모드 |

---

## 8. v3 없이 새 화면 보는 방법

`?v=3` 대신 아래 방법도 됩니다.

1. **강력 새로고침** (Mac Safari/Chrome: `Cmd + Shift + R`)  
2. **시크릿·사생활 보호** 모드로 열기  
3. 브라우저 **캐시·사이트 데이터 삭제** (해당 사이트만)  
4. 며칠 지나면 캐시가 자연히 갱신되기도 함 (느림)

**가장 빠른 확인:** `?v=3` URL 또는 시크릿 모드

## 10. 자주 묻는 질문

### Q. 카톡에는 항상 `?v=3`을 붙여야 하나?

**처음 한동안**은 붙이는 편이 안전합니다.  
나중에는 기본 URL만 써도 대부분 새 화면입니다.

### Q. `?v=3`을 붙이면 주소가 달라져서 문제 없나?

같은 사이트입니다. 검색·자리 찾기 기능은 동일합니다.

### Q. v4로 바꾸면 주소도 v4로?

네. 다음 큰 수정 때 `?v=4`, 코드 안도 `style.css?v=4`로 맞추면 됩니다.

---

## 11. 요약

| 질문 | 답 |
|------|-----|
| v3를 왜 쓰나? | **옛 CSS/JS 캐시**를 끊기 위해 |
| URL에 왜 붙이나? | **접속할 때마다** 새 파일을 받게 하려고 |
| 꼭 붙여야 하나? | 옛 화면이 보일 때 **확인용**으로 붙이면 됨 |
| 기본 URL은? | `https://kdc0901.github.io/web_location/` |

**제대로 반영 후 배포** = `git push` + (필요 시) **`?v=3`으로 확인**까지가 한 세트입니다.
