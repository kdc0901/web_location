## 1. 증상 (첨부 스크린샷 기준)

| 증상 | 설명 |
|------|------|
| 입력창 잘림 | 「이름으로 자리 찾기」 입력란이 오른쪽에서 잘림 |
| 책상 겹침 | 1행(1·2·3번)과 2행(4·5번) 칸이 **위아래로 겹침** |

## 2. 원인 정리

| 원인 | 설명 |
|------|------|
| PC 기준 CSS | 640px 이하에서만 모바일 → 일부 폰·캐시에서 2~3열 유지 |
| flex + scale | 검색 강조 시 `transform: scale` → 아래 행과 **겹침** |
| flex 행 배치 | `.desk-row` 높이 계산 오류로 2행이 1행 위에 겹침 |
| 가로 검색창 | 모바일에서도 input·버튼 가로 → **넘침** |
| 캐시 | `style.css`·Service Worker가 예전 버전 |

## 3. 해결 방향 (모바일 우선)

**기본 = 모바일(1열)** → 화면이 넓어질수록 2열·3열

| 화면 너비 | 두레 배치 | 검색창 |
|-----------|-----------|--------|
| ~599px | **1열** | 세로 (입력 → 버튼) |
| 600px~ | 1열 | 가로 (입력 + 버튼) |
| 768px~ | **2열** | 가로 |
| 1024px~ | **3열** | 가로 |

## 4. 핵심 코드 변경

### 4-1. 책상 배치 → CSS Grid (겹침 제거)

```css
.desks {
  display: grid;
  grid-template-columns: repeat(3, minmax(0, 1fr));
  gap: 0.4rem;
}

.desk-row {
  display: contents;  /* 행 래퍼 제거, 칸만 그리드에 배치 */
}
```

- 5명: 1·2·3행 / 4·5번은 2열에 배치 (`desk-row--pair`)
- 4명: 5번만 가운데 (`desk-row--single` → `grid-column: 2`)
- 6명: 4·5·6번이 다음 줄에 자동 배치

### 4-2. 모바일 책상 표시

- 번호 **위**, 이름 **아래** (세로)
- 강조 시 `scale` 대신 **테두리·배경색**만 사용 (겹침 없음)

### 4-3. 넘침 방지

```css
html, body, .app { overflow-x: hidden; }
.search-panel, .search-row input { width: 100%; max-width: 100%; }
```

### 4-4. 캐시 갱신

- `index.html`: `css/style.css?v=3`
- `sw.js`: `classroom-seating-v3`
- placeholder 짧게: `이름 입력`
