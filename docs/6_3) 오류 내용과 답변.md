# 오류 내용과 답변 (자리배치 · GitHub Pages)

자리배치 웹앱을 배포·공유할 때 나온 질문과 오류, 원인, 해결 방법을 정리한 문서입니다.

관련 문서: `6_2) (자리배치 파일)을 GitHub에 배포하는 절차.md`

---

## 1. localhost 주소를 카카오톡에 올리면 다른 사람도 보나?

### 질문

`http://localhost:8765/index.html` 주소를 카카오톡에 올리면, 다른 사람 휴대폰에서도 동일하게 index 화면이 보이는지?

### 답변

**아니요. 보이지 않습니다.**

| 주소 | 의미 | 다른 사람 폰 |
|------|------|-------------|
| `localhost` / `127.0.0.1` | **이 기기 자기 자신** | ❌ 접속 불가 |
| `https://kdc0901.github.io/web_location/` | 인터넷에 공개된 Pages | ✅ 접속 가능 |

- 친구 폰에서 `localhost`는 **친구 폰 자신**을 가리킵니다.
- 내 Mac에서 돌린 `python3 -m http.server`는 **내 Mac이 켜져 있을 때, 같은 네트워크**에서만 IP로 공유 가능합니다.

**카카오톡 공유용 권장 주소:**

`https://kdc0901.github.io/web_location/`

## 2. GitHub Pages 404 — There isn't a GitHub Pages site here

### 오류 화면 (내용)

브라우저에 다음과 비슷하게 표시됨:

- 큰 글씨 **404**
- **There isn't a GitHub Pages site here.**
- If you're trying to publish one, read the full documentation…

### 상황

- GitHub에 `web_location` 저장소는 있고 `git push`도 완료된 상태
- `https://kdc0901.github.io/...` 로 접속 시도
- 또는 문서 예시의 `classroom-seating` 같은 **다른 저장소 이름**으로 접속

### 원인

| 원인 | 설명 |
|------|------|
| **Pages 미설정** | 저장소 Settings → Pages에서 배포 소스(branch/folder)를 저장하지 않음 |
| **URL 오타** | 실제 저장소는 `web_location`인데 `classroom-seating` 등 다른 이름으로 접속 |
| **배포 대기** | Pages를 막 켠 직후 1~3분간은 404가 나올 수 있음 |
| **잘못된 경로** | `https://kdc0901.github.io/` 만 입력 (프로젝트 경로 없음) |

### 해결 (답변)

#### ① GitHub Pages 켜기

1. https://github.com/kdc0901/web_location 접속
2. **Settings** → **Pages**
3. 설정:
   - **Source**: Deploy from a branch
   - **Branch**: `main`
   - **Folder**: `/ (root)`
4. **Save**

1~3분 후 상단에 사이트 URL이 표시됩니다.

#### ② 올바른 주소로 접속

**정상 주소:**

- https://kdc0901.github.io/web_location/
- https://kdc0901.github.io/web_location/index.html

**404 나는 주소 예:**

| 주소 | 이유 |
|------|------|
| `https://kdc0901.github.io/classroom-seating/` | 저장소 이름 불일치 |
| `https://kdc0901.github.io/` | 프로젝트 Pages 경로 없음 |
| `http://localhost:8765/...` | 로컬 전용 |

#### ③ 여전히 안 될 때

1. Settings → Pages에서 상태가 **built** 인지 확인
2. 브라우저 **강력 새로고침**: Mac `Cmd + Shift + R`
3. 시크릿(사생활) 모드로 다시 열기
4. `index.html`이 저장소 **루트**에 있는지 GitHub에서 확인

### 조치 결과 (이번 프로젝트)

- Pages를 `main` / `(root)` 로 활성화함
- 빌드 상태: **built**
- 접속 확인: **HTTP 200** (정상)

## 3. 문서 예시 URL과 실제 URL이 다른 경우

### 오류/혼동

배포 가이드에 `classroom-seating` 예시가 있으나, 실제 저장소 이름은 **`web_location`**.

### 답변

GitHub Pages URL은 **저장소 이름**과 같아야 합니다.

```
https://[GitHub아이디].github.io/[저장소이름]/
```

이 프로젝트:

```
https://kdc0901.github.io/web_location/
```

## 4. 요약 표

| 증상 | 원인 | 해결 |
|------|------|------|
| 카톡에서 localhost 안 열림 | localhost는 본인 PC 전용 | GitHub Pages URL 공유 |
| 404 There isn't a GitHub Pages site | Pages 미설정 또는 URL 오류 | Settings → Pages 활성화 + `web_location` URL |
| CSS/JS 없이 HTML만 보임 | `css/`, `js/` 미업로드 | 루트에 폴더 포함 후 push |
| 예전 명단이 보임 | 브라우저 캐시 | 강력 새로고침 / 시크릿 모드 |
